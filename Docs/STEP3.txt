Implement Phase 2 Step 17 — Fullness buff MVP.

Goal:
Make FullnessMinutes actually affect work hunger cost.

Rules:
- No Workspace changes.
- No new zones.
- No monetization/combat/vehicles/trading.
- Keep Rojo scripts-only.
- Keep DataStore backward-compatible.
- Keep existing FoodInventory behavior.
- Do NOT implement passive hunger decay yet.
- Do NOT save fullness yet.

Current:
- Food items have FullnessMinutes.
- FoodInventory already exists.
- Eating food restores Energy/Hunger.
- FullnessMinutes is currently informational only.

Add session-only state:
- FullnessUntilGameMinutes
- Default 0
- Not saved to DataStore.

Implementation requirements:

1. Player stats/state
- Add a replicated value or attribute that the client can read:
  FullnessUntilGameMinutes
- Keep it session-only.
- Do not add it to DataStore.

2. Eating food
When EatFoodItem succeeds:
- Apply existing Energy/Hunger restore.
- Set FullnessUntilGameMinutes = current game minutes + item.FullnessMinutes.
- Handle wrap over midnight safely.
- Status message should include:
  "Full for <FullnessMinutes> min."

3. Fullness active check
Create clean server helper logic:
- Determine whether player is currently full based on current game minutes and FullnessUntilGameMinutes.
- Keep it robust around midnight.
- Server is source of truth.

4. Work hunger reduction
While fullness is active:
- Warehouse Shift HungerCost reduced by 50%.
  Example: 10 → 5.
- Cleanup Shift HungerCost reduced by 50%.
  Example: 5 → 3.
- Use consistent rounding: math.ceil(originalCost * 0.5).

5. Messages
Job complete messages should reflect actual hunger cost used.
Example:
"Warehouse Shift complete! +<pay> Wallet, -20 Energy, -5 Hunger, +10 XP."

6. HUD
Update JonesMoneyUi.client.lua:
- Show Fullness line:
  - "Fullness: None"
  - or "Fullness: Active"
- If easy, show remaining minutes:
  "Fullness: 75 min"
- Client only displays replicated state; server decides effects.

Update:

- Docs/JONES_FOOD_SHOP.md
- Docs/JONES_ROBLOX_ADAPTATION.md

Document:
- Fullness is session-only
